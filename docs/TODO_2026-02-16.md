## Důležité pro vydání

- [ ] **OTA při prvním releasu (Jobi)** – viz checklist níže. Build: `npm run tauri:build:universal:signed` (načte klíč z `~/.tauri/jobi.key`). Viz `docs/OTA_SETUP_KROKY.md`.
  - **Checklist po buildu:**  
    1. V kořeni projektu je **latest.json** (vygenerovaný skriptem).  
    2. V `src-tauri/target/release/bundle/macos/` jsou **jobi.app.tar.gz** a **jobi.app.tar.gz.sig**.  
    3. Na GitHubu (repo jobi) vytvoř **Release** (tag např. `v0.1.0`, shodný s `version` v `tauri.conf.json`).  
    4. Do release nahraj tyto 3 soubory: **latest.json**, **jobi.app.tar.gz**, **jobi.app.tar.gz.sig**.  
    5. Od té chvíle aplikace s tímto endpointem bude umět stahovat OTA updaty.
- [ ] **OTA při prvním releasu (JobiDocs)** – v `jobidocs/` nastavit **GH_TOKEN** a (pro notarizaci) **APPLE_ID**, **APPLE_APP_SPECIFIC_PASSWORD**, **APPLE_TEAM_ID**, pak `npm run electron:build`. Podpis (Developer ID) a hardenedRuntime jsou v `electron-builder.json`. Viz `docs/JOBIDOCS_OTA_A_RELEASE.md`.
- [ ] **Apple Developer – podpis a notarizace (Jobi)** – účet zaplacen. Další kroky:  
  1. V **Apple Developer** (developer.apple.com) vytvoř **Developer ID Application** certifikát (Certificates → + → Developer ID Application).  
  2. Stáhni certifikát, nainstaluj do Keychain.  
  3. V **Tauri:** v `src-tauri/tauri.conf.json` v `bundle.macOS` nastav **signingIdentity** na jméno certifikátu (např. `"Developer ID Application: Tvé Jméno (TEAM_ID)"`).  
  4. Při buildu pak bude .app podepsaný – uživatel nebude muset „povolit neidentifikovaného vývojáře“.  
  5. **Notarizace (doporučeno):** po buildu zabalit .app do .dmg nebo .zip, poslat na Apple (`xcrun notarytool submit`), po schválení „staple“ (`xcrun stapler staple`). Pak distribuce .dmg/.zip – Gatekeeper na jiných Macích nebude blokovat. Viz `docs/INSTALACE_KLIENTUM.md` a [Tauri – macOS code signing](https://tauri.app/distribute/sign/macos/).
6. **OTA s notarizací:** při každém releasu: build → notarize (`./scripts/notarize-jobi.sh`) → **`./scripts/pack-notarized-ota.sh`** (zabalí notarizovanou .app do .tar.gz, podepíše, vygeneruje latest.json) → nahrát 3 soubory na GitHub Release.
- [ ] **Instalace klientům – podepisování** – viz položka „Apple Developer – podpis a notarizace“ výše. Bez podpisu musí klient na Macu povolit „neidentifikovaného vývojáře“ (pravý klik → Otevřít).
